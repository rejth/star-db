Основные этапы жизненного цикла компонента и их методы:

1. MOUNTING
constructor() -> render() (преобразование JSX-элементов в DOM-объекты -> встраивание DOM-объектов в DOM-дерево) -> componentDidMount()
componentDidMount():
  a. вызывается, когда компонет уже подключен к DOM-дереву, вмонтирован в него, т.е элементы уже видны на странице
  a. используется для инициализации (получение данных с сервера, работа с DOM)

2. UPDATES
New Prop или setState() -> componentDidUpdate(prevProps, prevState)
componentDidUpdate():
  a. вызывается после того, как компонент обновился
  b. компонент обновляется после того, как получил новые свойства
  с. этот метод вызывается после render() - в нем можно, к примеру запрашивать новые данные для обновленных свойств.

3. UNMOUNTING
Удаление компонента из DOM-дерева
componentWillUnmount() - полезен для удаления слушателей событий, таймеров или асинхронных запросов после удаления компонента из DOM

4. ERROR
Компонент получает ошибку, которая не была поймана ранее
componentDidCatch() - особенно полезен для продакшена,
потому что в продакшене бывают ситуации, когда в случае ошибки приложение не выбрасывает никакого уведомления
и пользователь видит пустой экран. В этом случае полезно обработать ошибку и показать уведомление
работает только в ошибках в методах жизненного цикла! Не будет вызываться в асинхронных вызовах, даже если этот вызов есть в методе жизненного цикла, это не замена валидации данных и другой обработки ошибок!
